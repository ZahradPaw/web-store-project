End-поинты:
-api/v1/auth/login - вход в систему по сессии (убрать потом)
-api/v1/auth/logout - выход из системы по сессии (убрать потом)
-api/v1/auth/token - вход по JWT
-api/v1/auth/token/refresh - refresh JWT
-api/v1/users/register - регистрация (незарег.)
-api/v1/users/list - список пользователей (продавец только клиентов, админ всех пользователей)
-api/v1/users/profile - профиль текущего пользователя, только просматривать (зарег.)
-api/v1/users/user/<int:pk> - параметры и редактирование пользователя с указанным id (админ)
-api/v1/products - список продуктов (каждый смотреть, товаровед и админ добавлять новые)
-api/v1/products/<int:pk> - параметры продукта (каждый смотреть, админ и товаровед редакт.)
-api/v1/orders - список заказов (клиент только свои, может добавить от себя, админ и продавец видят все)
-api/v1/orders/?client=1 - можно указать id клиента для получения его заказов (для админа и продавца)
-api/v1/orders/<int:pk> - редактирование заказа (админ и менеджер, только дату доставки и статус)
-api/v1/orders/<int:pk>/cancel - отмена заказа (клиент свои заказы, менеджер по закупкам и админ все)
-api/v1/orders/<int:pk>/mark_confirmed - заказ подтверждён (только менеджер по закупкам и админ)
-api/v1/orders/<int:pk>/mark_delivered - заказ доставлен (только менеджер по закупкам и админ)
-api/v1/loyalty - текущие настройки лояльности (все смотреть могут, админ редакт.)
