# Веб-приложение интернет-магазина

Fullstack-проект интернет-магазина с разделением на backend на Django REST Framework и frontend на ReactJS. 
В данном проекте реализован функционал оформления заказов различных товаров, а также разделение на различных типов пользователей.

## Описание

Без авторизации в системе на сайте доступен только просмотр каталога товаров, а также регистрация в системе. Зарегестрированный пользователь имеет роль покупателя.
Пользователи разделены на покупателей, сотрудников и администратора:

* **покупатель**: просмотр каталога товаров, оформление и просмотр своих заказов.
* **товаровед**: просмотр, добавление, редактирование и удаление товаров.
* **клиент-менеджер**: регистрация, просмотр всех покупателей и редактирование их данных. 
* **продавец**: оформление и управление статусами всех заказов.
* **директор**: просмотр таблиц всех товаров, покупателей и заказов с установкий различных фильтров.
* **администратор**: доступ ко всем данным системы. 

Каждый заказ имеет определенный статус (оформлен, оплачен, доставлен, отменен), дату оформления и дату доставки, которую можно изменить.

Также на сайте реализован функционал настраиваемой программы лояльности, позволяющий давать покупателям скидки на заказы при достижении определенной суммы покупок. 

## Backend и API

Сервис имеет открытый RESTful API для получения и изменения данных.

Основные end-поинты:
```
-api/v1/auth/token - вход по JWT-токену
-api/v1/auth/token/refresh - обновлениие JWT-токена
-api/v1/users/register - регистрация 
-api/v1/users/list - список пользователей 
-api/v1/users/list/?name=Иван&role=client - фильтрация по имени пользователя и его роли
-api/v1/users/profile - профиль текущего пользователя, только просматривать 
-api/v1/users/user/<int:pk> - параметры и редактирование пользователя с указанным id 
-api/v1/products - список продуктов
-api/v1/products/?name=Яблоко - фильтрация по названию товара
-api/v1/products/<int:pk> - параметры продукта 
-api/v1/orders - список заказов 
-api/v1/orders/?client_id=1 - можно указать id клиента для получения его заказов 
-api/v1/orders/?client_name=Иван - фильрация заказов по имени покупателя
-api/v1/orders/<int:pk> - редактирование заказа 
-api/v1/orders/<int:pk>/cancel - отмена заказа 
-api/v1/orders/<int:pk>/mark_paid - заказ оплачен
-api/v1/orders/<int:pk>/mark_delivered - заказ доставлен
-api/v1/loyalty - текущие настройки лояльности 
```

## Развертывание с Docker Compose

Для полноценного развертывания веб-сервиса на сервере необходимо установить **docker** и **docker compose**. Также необходимо определить параметры окружения в **.env** файлах в главном каталоге, где указываются параметры backend и базы данных, а также **.env** файл в каталоге frontend, где можно указать свой URL для API-запросов. Заполнять на основе примеров из **.env.example**. 


После настроек параметров можно запустить сервис через команду **docker compose**:

```
docker compose up --build
```

После этого сайт будет доступен по http://localhost, порт 80. API сервиса доступен по маршруту http://localhost/api/v1. Также доступна админ-панель Django по маршруту http://localhost/admin.

Для добавления администратора в сервис необходимо выполнить команду ниже, после чего ввести логин, адрес почты и пароль.

```
docker compose exec backend python manage.py createsuperuser
```

Для остановки нужно выполнить команду остановки контейнеров:

```
docker compose down
```

В папке logs будут доступны файлы со всеми логами **access.log** и **error.log**. 

В проекте также доступна фикстура образца базы данных. Для того, чтобы загрузить в БД все данные из образца, необходимо выполнить команду:

```
docker compose exec backend python manage.py loaddata db_example.json
```

После загрузки образца можно автоизоваться на сайте, как администратор, используя логин **root** и пароль **root12345** (для этого также нужно, чтобы параметр .env DJANGO_SECRET_KEY был, как в примере). Для доступа к учетным записям других пользователей использовать их логины или почту и пароли, которые совпадают с логинами.
